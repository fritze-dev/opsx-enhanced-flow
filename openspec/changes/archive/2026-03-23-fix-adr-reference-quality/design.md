# Technical Design: Fix ADR Reference Quality

## Context

An audit of 12 consolidated ADRs generated by `/opsx:docs` revealed three categories of reference quality issues: broken spec links (1 ADR referencing a renamed spec), external GitHub URLs with wrong org name (3 ADRs, 7 URLs), and missing cross-references between related ADRs (6 ADRs). The root cause for the URL problem is that external links don't belong in ADR References — the archive backlink already provides traceability to GitHub issues via the archive's proposal.md.

## Architecture & Components

| File | Change Type | Description |
|------|------------|-------------|
| `skills/docs/SKILL.md` | Modified | Restrict References to internal links only, add validation and cross-reference heuristic |
| `openspec/specs/decision-docs/spec.md` | Modified (via sync) | Internal-only references, validation, cross-reference requirements |

### SKILL.md Changes (Step 4, References paragraph)

Modify the existing References instructions:

1. **Internal links only**: Replace the current instruction that includes GitHub issue links with: "References SHALL contain only internal relative links: archive backlinks (`[Archive: ...]`), spec links (`[Spec: ...]`), and ADR cross-references (`[ADR-NNN: ...]`). Do NOT include external URLs (GitHub issues, external docs). The archive backlink provides traceability to issues via the archive's proposal.md."

2. **Reference validation**: "After writing each ADR's References, validate all links:
   - For `[Spec: <name>]` links: glob `openspec/specs/<name>/spec.md` — if it doesn't exist, replace with successor spec(s) or add `<!-- REVIEW -->` marker.
   - For `[Archive: <name>]` links: glob `openspec/changes/archive/*-<name>/` — verify the archive exists."

3. **Cross-reference heuristic**: "Check if the current ADR's archive modifies a system established by an earlier ADR (look for explicit references to other changes in proposal.md/design.md, or overlapping specs/ subdirectories with earlier archives). If a clear relationship exists, add a cross-reference to the earlier ADR."

## Goals & Success Metrics

* No ADR References section contains external URLs (GitHub, http://, https://) — PASS if zero external URLs in any generated ADR
* Every `[Spec: ...]` link in generated ADRs points to an existing spec — PASS if `openspec/specs/<name>/spec.md` exists for every linked name
* Every `[Archive: ...]` link points to an existing archive directory — PASS if archive dir exists for every link
* ADRs that modify a system established by an earlier ADR include a cross-reference — PASS if ADR-012 references ADR-003

## Non-Goals

- Automated dependency graph between all ADRs
- Retroactive fixing of already-generated ADRs (done manually in previous commit)

## Decisions

| # | Decision | Rationale | Alternatives Considered |
|---|----------|-----------|------------------------|
| 1 | Internal-only references in ADRs | Eliminates external URL maintenance entirely. Traceability to issues preserved via archive backlink → proposal.md → issue reference. | GitHub URL normalization from plugin.json (still requires URL maintenance); keep external URLs with validation (doesn't eliminate root cause) |
| 2 | Post-generation reference validation via glob | Agent can verify spec and archive links exist before writing; catches renames/splits | Manual review only (error-prone, not scalable) |
| 3 | Heuristic cross-referencing (explicit mentions + overlapping specs) | Catches obvious relationships without over-linking; conservative by default | Full graph analysis (too complex for prompt); no cross-refs (quality degrades over time) |

## Risks & Trade-offs

- **Less direct traceability to GitHub issues** → Mitigation: Archive backlink provides one-hop traceability (ADR → Archive → proposal.md → issue). Acceptable trade-off for eliminating URL maintenance.
- **Agent may miss cross-references** → Mitigation: Conservative heuristic means false negatives (missing link) not false positives (wrong link). Manual review can add missing links.
