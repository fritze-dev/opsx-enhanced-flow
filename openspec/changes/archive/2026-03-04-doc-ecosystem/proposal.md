## Why

`/opsx:docs` reads only baseline specs, leaving valuable context from archived artifacts (proposal, research, design, preflight) unused. Users can't understand *why* capabilities exist, what alternatives were considered, or what limitations apply — without digging through archives manually. Additionally, the `docs-generation` capability is too broad — it mixes user docs, changelog, and now would gain architecture overview and ADRs. This change splits it into focused capabilities and adds the new doc types.

## What Changes

- **Capability split:** `docs-generation` is removed and split into `user-docs` (enriched capability docs), `architecture-docs` (architecture overview), and `decision-docs` (ADRs). Changelog generation moves to `release-workflow` where it logically belongs.
- **Enriched capability docs:** When generating a capability doc, `/opsx:docs` looks up `archive/*/specs/<capability>/` to find relevant archives, then reads their proposal, research, design, and preflight artifacts to add "Why This Exists", "Background", and "Known Limitations" sections
- **Architecture overview:** New cross-cutting document `docs/architecture-overview.md` synthesized from constitution, three-layer-architecture spec, and all design.md Decisions tables
- **ADR generation:** Each row from `## Decisions` tables across all archived design.md files becomes a formal ADR at `docs/decisions/adr-NNN-slug.md` with index. ADR "Context" sections are enriched with research.md data (approaches, findings) where available.
- **Expanded TOC:** `docs/README.md` updated to link architecture overview and decision records sections

## Capabilities

### New Capabilities

- `user-docs`: Enriched user-facing capability documentation generated by `/opsx:docs` — includes archive-derived sections (Why This Exists, Background, Known Limitations)
- `architecture-docs`: Architecture overview document synthesized from constitution, specs, and design decisions
- `decision-docs`: Architecture Decision Records (ADRs) generated from archived design.md Decisions tables

### Modified Capabilities

- `release-workflow`: Gains the changelog generation requirement (moved from `docs-generation`)

### Removed Capabilities

- `docs-generation`: Split into `user-docs`, `architecture-docs`, `decision-docs`, with changelog moved to `release-workflow`

## Impact

- `skills/docs/SKILL.md` — Extended with new steps (archive lookup, enrichment extraction, architecture overview, ADR generation) and enriched template

- `docs/decisions/*.md` — New ADR files (with integrated research context) + index

## Scope & Boundaries

**In scope:**
- Splitting `docs-generation` into focused capabilities
- Enriching `/opsx:docs` skill prompt
- Archive lookup per capability via glob (`archive/*/specs/<capability>/`)
- Enriched capability doc template with conciseness guards
- Architecture overview generation
- ADR generation from design.md Decisions tables, enriched with research.md context
- Moving changelog to `release-workflow`
- TOC expansion

**Out of scope:**
- `/opsx:changelog` behavior changes (unchanged, just moves to different spec)
- New skills or subcommand flags (everything in `/opsx:docs`)
- tasks.md usage in docs (implementation-internal, no user value)
- Schema or constitution modifications
