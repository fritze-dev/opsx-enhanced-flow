# Technical Design: Visible Assumptions & REVIEW Auto-Resolution

## Context

Assumption markers in specs use two patterns inconsistently: Pattern A (visible list item + hidden HTML tag) and Pattern B (everything hidden). Pattern B was a formatting error that makes assumptions invisible in Markdown preview. Separately, `<!-- REVIEW -->` markers placed by bootstrap and docs skills are never actively resolved — they sit invisibly in files.

## Architecture & Components

**Files affected:**

1. **Schema instructions** (`openspec/schemas/opsx-enhanced/schema.yaml`):
   - Line 142: specs artifact instruction — prescribe Pattern A format
   - Line 169: design artifact instruction — prescribe Pattern A format

2. **Templates** (`openspec/schemas/opsx-enhanced/templates/`):
   - `specs/spec.md`: assumption section guidance → Pattern A
   - `design.md`: assumption section guidance → Pattern A
   - `preflight.md`: marker collection guidance → audit both ASSUMPTION and REVIEW

3. **Baseline specs** (`openspec/specs/` — 11 specs with Pattern B):
   - Delta specs convert all Pattern B → Pattern A in `## Assumptions` sections
   - Inline Pattern B assumptions moved to `## Assumptions` section

4. **Skills**:
   - `skills/bootstrap/SKILL.md`: Add REVIEW auto-resolution loop (iterate markers, ask user, document, delete)
   - `skills/docs/SKILL.md`: Add REVIEW auto-resolution for broken reference validation (resolve links, ask user if needed, no markers in output)

5. **Constitution** (`openspec/constitution.md`):
   - Code Style: Update marker convention to reference Pattern A format

## Goals & Success Metrics

- PASS/FAIL: Zero Pattern B assumptions (`<!-- ASSUMPTION:` with no preceding visible text) remain in baseline specs
- PASS/FAIL: Zero `<!-- REVIEW` markers remain in non-archived files after skills complete their runs
- PASS/FAIL: Preflight Section F ("Marker Audit") audits both ASSUMPTION format + REVIEW presence
- PASS/FAIL: Bootstrap skill actively resolves all REVIEW markers before finishing
- PASS/FAIL: Docs skill produces ADRs with zero REVIEW markers

## Non-Goals

- Migrating archived changes (immutable)
- Changing REVIEW marker format (stays as HTML comment — it's transient)
- Adding automated linting or CI checks for marker format
- Updating generated docs under `docs/` (regenerated by `/opsx:docs` after archive)

## Decisions

| Decision | Rationale | Alternatives |
|----------|-----------|--------------|
| Enforce Pattern A (`- Visible text. <!-- ASSUMPTION: tag -->`) | Preserves machine-parseable tag for preflight while making assumption visible to reviewers | Blockquote format — loses machine tag; Pattern B enforcement only via preflight — doesn't fix existing content |
| REVIEW stays as HTML comment, resolved automatically | REVIEW is transient (exists only during agent session), so visibility in final output doesn't matter — what matters is active resolution | Make REVIEW visible too — unnecessary since it should never persist |
| Delta specs for all 13 affected baseline specs | Verify can check correctness; sync handles the merge; consistent with project workflow rules | Direct edits during apply — violates schema constraint on baseline edits |
| Preflight expanded to "Marker Audit" (Section F) | Single audit covering both marker types; REVIEW markers are always Blocking since they must be resolved | Separate sections for each marker type — unnecessary complexity |

## Risks & Trade-offs

- **Large number of delta specs (13)** → All are mechanical format changes with minimal risk; verify catches any errors
- **Inline Pattern B assumptions moved to Assumptions section** → Loses inline proximity to context, but gains auditability in one place
- **REVIEW auto-resolution adds interactive prompts** → May slow down bootstrap/docs runs, but ensures no invisible markers persist

## Assumptions

- No in-progress changes use the old Pattern B format (verified: git status is clean). <!-- ASSUMPTION: Clean starting state -->
- The OpenSpec CLI does not parse or depend on assumption or review markers — this is purely an agent/human convention. <!-- ASSUMPTION: CLI marker independence -->
