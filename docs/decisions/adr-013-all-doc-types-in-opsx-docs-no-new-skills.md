# ADR-013: All doc types in /opsx:docs, no new skills

## Status
Accepted (2026-03-04)

## Context
With the addition of architecture overviews and ADR generation to the documentation ecosystem, there was a choice between creating separate skills (`/opsx:adr`, `/opsx:research-log`) or consolidating all documentation types under the existing `/opsx:docs` skill. The user expressed a clear preference for avoiding skill proliferation. Research showed that all documentation types share the same source materials (specs, archives) and output location (`docs/`), making them natural candidates for a single entry point. Running `/opsx:docs` regenerates everything in one pass, ensuring consistency across all documentation outputs.

## Decision
All documentation types (capability docs, architecture overview, ADRs) are generated by `/opsx:docs` as a single entry point. No new skills are created.

## Rationale
User preference to avoid skill proliferation. A single entry point is simpler and ensures consistency across all documentation outputs.

## Alternatives Considered
- Separate `/opsx:adr` and `/opsx:research-log` skills — adds maintenance burden, requires users to remember multiple commands

## Consequences

### Positive
- Single command generates all documentation consistently
- No additional skills to maintain or discover
- Simpler user experience with one entry point

### Negative
- Longer skill prompt with more complex step sequence; mitigated by clear section headers
